# Wedding Invitation

## Mobile-only режим

Публичная страница `index.html` — это минимальная оболочка. Она показывает исходное приглашение в `iframe` шириной не более `430px`:

- на телефоне используется вся ширина экрана;
- на широком экране приглашение остаётся мобильным и центрируется;
- исходная Tilda/Zero Block-разметка перенесена без изменений в `invitation.html`;
- `invitation.html` запускается как отдельная Vite-страница, поэтому её скрипты видят реальную мобильную ширину viewport, а не ширину монитора.

Такой подход не меняет Tilda ID, абсолютные координаты, блоки или медиа исходного мастер-макета.

## Где менять данные пары

1. Обновите `wedding.json`.
2. Запустите `npm run build` — перед сборкой автоматически выполняется `scripts/generate-wedding-images.mjs`.
3. Опубликуйте содержимое `dist/`.

`src/main.js` подставляет данные из `wedding.json` в неизменный шаблон. Не меняйте геометрию `invitation.html`, если задача относится только к данным новой пары.

## Локальная проверка

```bash
npm install
npm run build
npm run preview -- --host 0.0.0.0
```

Откройте адрес preview. Для проверки мобильного шаблона используйте ширины `360`, `375`, `390`, `412` и `430px`; на более широких экранах canvas должен быть ровно `430px` и находиться по центру.
